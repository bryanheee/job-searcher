GitHub Connection
Security notice: Your Personal Access Token (PAT) is stored only in your browser's localStorage and is never committed to the repository or sent anywhere except directly to the GitHub API (api.github.com). Clear it any time with the "Clear saved token" button below.
Token is stored only in this browser — never in the repo.
How to create a PAT with the right permissions:
  1. Go to github.com/settings/tokens (fine-grained tokens recommended)
  2. Click Generate new token
  3. Set expiry (90 days is reasonable), select your repository under Repository access
  4. Under Repository permissions grant:
    • Contents — Read and write (to update config.json)
    • Actions — Read and write (to trigger workflow_dispatch)
  5. Click Generate token, copy it, and paste it above
Treat the PAT like a password. Never paste it into any file in your repo.
Search Queries
One query per line — sent to both Indeed NL and LinkedIn
Keyword Tiers (weight × unique matches = score contribution)
One keyword/phrase per line per tier. Score contribution = weight × number of unique keywords matched in that tier. Add tiers with any weight; remove tiers you don't need (at least one must remain).
Blacklists
One term per line
Seniority Filter Patterns
Regex patterns applied to job titles only — one per line. Jobs whose title matches any of these are excluded (fresh grad filter).
Target Companies
One company name per line (lowercase). Matching jobs are flagged with a star and sorted to the top of results.
